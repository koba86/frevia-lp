# 学びの記録（Lessons Learned）

## 2026-03-14: DNS設定前にネームサーバーを確認すること

### 何が起きたか
- お名前.comで購入したドメインに対して、お名前.comのDNS管理画面でCNAMEレコードを追加した
- しかし実際のネームサーバーはCloudflareに変更されていた
- 結果、48時間以上経ってもDNSが反映されなかった

### 原因
- 「お名前.comで買ったドメイン ＝ お名前.comのネームサーバーを使っている」と確認せずに決めつけた
- DNS設定前に `dig ドメイン名 NS` でネームサーバーを確認しなかった

### 今後のルール
- **DNS設定を案内する前に、必ず `dig ドメイン名 NS +short` を実行してネームサーバーを確認する**
- ネームサーバーがドメイン登録業者と異なる場合は、ユーザーに説明して正しい場所で設定するよう案内する
- 思い込みで手順を案内しない。必ず現状を確認してから案内する
