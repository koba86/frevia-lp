# frevia-lp 仕様書

作成日: 2026-07-03

## 1. 目的
配信事務所Freviaの「ライバー（配信者）募集」ランディングページ（LP）。
訪問者に事務所の魅力を伝え、ページ内の応募フォームから応募してもらうことがゴール。

## 2. 概要
- 静的1ページの構成（HTMLファイル1枚に、案内文・画像・応募フォーム・アニメーションが
  すべて収まっている）。フレームワークやビルド作業は使っていない。
- 公開場所: GitHub Pages（GitHubの無料ホスティング機能）。
- 公開URL: `https://recruit.frevia-agency.jp`
  （`CNAME` ファイルにこの独自ドメインが設定されている）
- 公開方法: `master`ブランチへ変更を push（アップロード）すると、GitHub Actions
  （GitHubの自動処理の仕組み）が自動的にサイトへ反映する。手作業のデプロイ操作は不要。
  設定ファイル: `.github/workflows/`

## 3. ファイル構成
- `index.html` … ページ本体（HTML・CSS・JavaScriptがすべてこの1ファイルに含まれる）
- `CNAME` … GitHub Pagesに独自ドメインを紐づけるための設定ファイル
- `favicon.png` … ブラウザタブに表示される小さいアイコン
- `images/` … ページ内で使う画像素材
- `robots.txt` / `sitemap.xml` … 検索エンジン向けの設定
- `.github/workflows/` … GitHub Pagesへの自動公開設定
- `tasks/PROJECT_SPEC.md` … この仕様書
- `tasks/lessons.md` … 学びの記録
- `tasks/todo.md` … 作業ごとの一時的なチェックリスト（プロジェクト全体の恒久ルールではない）

## 4. ページ構成（index.html内のセクション、上から順）
| id | 内容 |
|---|---|
| `hero` | ファーストビュー（キャッチコピー「あなたの『好き』が、誰かの『推し』になる。」＋粒子アニメーション演出） |
| `target` | どんな人に向いているかの案内 |
| `benefits` | Freviaが提供する4つのサポート内容 |
| `record` | 実績の数字紹介 |
| `trust` | 運営体制の透明性（会社情報など） |
| `entry` | 応募の流れの案内＋応募フォームを開くボタン |

応募フォーム自体は上記セクションの外、モーダルウィンドウ（ボタンを押すと前面に
ポップアップ表示される小窓）として実装されている（`#modalOverlay` 内の `#entryForm`）。
このほか、プライバシーポリシー・利用規約を表示する法務モーダル（`#legalOverlay`）もある。

## 5. 応募フォームの仕様

### 入力欄（合計7項目＋ハニーポット1項目）
| 項目名（name属性） | ラベル | 必須 | 補足 |
|---|---|---|---|
| name | お名前 | 必須 | |
| age | 年齢 | 必須 | 数値、13〜99の範囲指定あり |
| email | メールアドレス | 必須 | |
| gender | 性別 | 任意 | 選択式（女性／男性／その他／回答しない） |
| experience | 配信経験 | 任意 | 選択式（未経験／半年未満／半年〜1年／1年以上） |
| tiktok | TikTok URL | 任意 | |
| pr | 自己PR | 任意 | 自由記述 |
| website | （非表示・ハニーポット） | - | 下記参照 |

- **ハニーポット欄**: `name="website"` の入力欄で、CSSにより画面上には表示されない
  （`aria-hidden="true"` かつ視覚的に隠すスタイル、`tabindex="-1"` でキーボード操作からも除外）。
  人間の利用者は入力しないため空のまま送信される。自動送信するスパムプログラムは
  この欄にも機械的に値を入れがちなので、値が入っていればスパムとみなして
  バックエンド側で静かに弾く仕組み（詳細は frevia-recruit-api 側の仕様書）。

### 送信の仕組み（handleSubmit関数、index.html内）
- 送信先: `https://recruit-api.frevia-agency.jp/api/apply`（バックエンドAPI、POST・JSON形式）
- 送信中はボタンを無効化し、表示テキストを「送信中…」に変える（連打防止）。
- タイムアウト: 15秒。応答がなければ通信失敗として扱う。
- **成功時**: 「ご応募ありがとうございます！担当者（info@frevia-agency.jp）より
  追ってご連絡いたします。」とアラート表示 → フォームを閉じて入力内容をリセットする。
- **失敗時**（通信エラー・タイムアウト・サーバー側エラーいずれの場合も）:
  サーバーから返ってきたエラーメッセージ（あれば）に加えて「送信できませんでした。
  お手数ですが、info@frevia-agency.jp へ直接メールでご応募ください。」とアラート表示。
  この時、**入力済みの内容はフォーム上に残したまま**にする（reset()を呼ばない）ため、
  応募者が入力し直す手間を減らせる。

## 6. バックエンド（応募の受付処理）について
応募フォームが送信したデータを実際に受け取り、保存・通知メール送信を行うのは
このリポジトリの外にある別プロジェクト。

- 場所: `~/Claude/apps/frevia-recruit-api`
- 詳しい仕様（API仕様・データベース構造・通知メールの内容・エラーコードなど）は
  `~/Claude/apps/frevia-recruit-api/tasks/PROJECT_SPEC.md` を参照。
- 応募データの確認方法や運用コマンドも同リポジトリの `CLAUDE.md` を参照。
